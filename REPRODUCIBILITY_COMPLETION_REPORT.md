# Reproducibility Upgrade Report

Completion date: 2026-06-25

## What the Repository Can Reproduce

### Level 1: Paper Tables and Figures

The repository rebuilds paper-facing Markdown tables and the same-budget
controls plot from canonical cleaned CSV files:

```bash
python scripts/build_paper_tables.py
python scripts/build_figures.py
```

### Level 2: Case-Level Data Audit

Sanitized case-level bundles are included for:

- Qwen3-4B
- Qwen2.5-3B
- Qwen2.5-14B
- Llama3.2-3B
- Gemma2-9B boundary evidence

The audit recomputes:

- Qwen3-4B: 72/72
- Qwen2.5-3B: 72/72
- Qwen2.5-14B: 14/14
- Llama3.2-3B: 25/25
- Total main: 183/183
- Gemma2 key-only Top-k: 7/72
- Gemma2 Uniform K6/V2: 7/72
- Gemma2 Uniform K6/V4: 72/72

Each bundle contains discovery/classification data, sensitive cases, risk
ranking, Top-k rows, same-budget controls, first-success cases, efficiency
summary, and source hashes.

### Level 3: Model Execution Interface

The repository provides:

- `src/tbgmp/kv_cache_wrapper.py`: external backend contract;
- `src/tbgmp/quantization.py`: precision policy and nominal saving utilities;
- `experiments/run_full_pipeline.py`: discovery, sensitive mining,
  Top1--Top12, and matched Random/Bottom orchestration;
- generic policy/path/model configs;
- an adapter template;
- sanitized Stage A, Top-k, and full-pipeline Slurm templates.

Users must supply model weights, CUDA/PyTorch/Transformers, and a compatible
TurboQuant/KV-cache backend.

## What the Repository Cannot Reproduce Alone

- It does not contain model checkpoints.
- It does not ship a universal quantizer kernel.
- It does not include raw cluster logs or complete generations.
- It does not guarantee one backend works across every model/cache API.
- Supporting models other than Gemma2 currently remain at cleaned summary
  level because suitable full case-level source bundles were not included in
  the release inputs.

Four main-model bundles are stored below `results/main_evidence/<model>/`.
Gemma2 is stored below `results/supporting/gemma2_9b/`.

The files were generated by `scripts/sanitize_case_level_sources.py` from local
experiment exports using a strict field whitelist. Full responses, hosts,
devices, partitions, timestamps, local paths, and raw log references were
removed. Short response excerpts are limited to 80 characters.

## Remaining Gaps

The primary remaining gap is the external model backend. The public interface
is executable and testable, but full GPU reruns require the original or a
compatible TurboQuant integration. Qwen2.5-14B risk ranking is published as
`policy_order_only` because a separate numeric risk-score export was not
available; no risk values were invented.
